# 功能名称：邮件监控与结构化归档 MVP 任务拆分

## 一、目标
本文件用于给前后端分别安排开发任务，作为当前 MVP 的执行拆分基线。所有任务以 [mail-monitoring-mvp.md](D:/Projects/codex/mail_ds_monitor/docs/features/mail-monitoring-mvp.md) 和 [mail-monitoring-mvp.yaml](D:/Projects/codex/mail_ds_monitor/docs/contracts/mail-monitoring-mvp.yaml) 为准。

## 二、后端开发任务
### P0
- 完成配置加载、数据库连接、基础日志
- 完成 `mailboxes` 表、`mail_messages` 表、`archive_records` 表、`summary_configs` 表、`summary_send_records` 表、`task_logs` 表
- 完成邮箱配置接口：列表、创建、更新
- 完成 IMAP 拉取服务：按邮箱拉取邮件、解析正文、保存原始记录
- 完成邮件幂等规则：`mailbox_id + internet_message_id` 和 `mailbox_id + provider_uid`
- 完成 AI 提取客户端封装和归档写入
- 完成归档列表和详情接口

### P1
- 完成汇总配置接口：列表、创建、更新
- 完成汇总手动发送接口和发送记录查询接口
- 完成 SMTP 发信封装
- 完成任务日志、失败日志和重试策略

### P2
- 完成更多字段提取、AI Prompt 版本管理和模型调用审计

## 三、前端开发任务
### P0
- 初始化 React 工程骨架、路由、状态和 API 封装
- 完成邮箱管理页面：列表、新增、编辑、启停
- 完成归档列表页面：分页、邮箱筛选、时间筛选、状态筛选
- 完成归档详情页面：邮件原文、AI 提取结果、失败原因展示

### P1
- 完成汇总配置页面：收件人、发送时间、邮箱范围、空结果策略
- 完成汇总发送记录页面：状态筛选、失败原因、时间窗口展示
- 完成全局异常处理、空态、加载态和基础权限控制

### P2
- 完成交互优化、批量操作、页面可观测性埋点

## 四、联调顺序
1. 先联调邮箱配置接口
2. 再联调归档列表和详情接口
3. 最后联调汇总配置和汇总发送接口

## 五、验收标准
- 前后端字段命名、枚举值、分页结构与契约一致
- 邮箱配置可实际保存到 MySQL
- IMAP 拉取的原始邮件可保存到 MySQL
- AI 提取结果可查询、可追踪失败原因
- 每日汇总可手动触发并记录发送结果

## 六、风险清单
- 邮件格式复杂，正文解析结果不稳定
- AI 返回结构漂移可能导致前端字段展示异常
- 多邮箱并发拉取和重复触发会带来幂等和任务冲突
- SMTP 发送受外部网络和服务商限制影响较大

## 七、待确认问题
- AI 提取字段清单是否固定
- 汇总无数据时是否跳过发送
- 权限模型首期是否只区分管理员和查看者
- 前端首期是否需要登录态页面
