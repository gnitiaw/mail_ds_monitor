# 邮箱管理操作列布局优化

## 背景
- 邮箱管理页当前把 `编辑 / 查看原始邮件 / 拉取 / 处理已拉取邮件` 全部平铺在列表操作列。
- 操作文案较长，导致操作列宽度失控，表格内容被横向撑破，影响可读性和点击体验。

## 目标
- 保留现有邮箱管理能力，不减少任何已有操作。
- 优化操作列布局，避免操作按钮把表格撑出可视区域。
- 让高频操作优先可见，次级操作收纳但仍然可达。

## 非目标
- 不调整邮箱拉取、处理、编辑的业务逻辑。
- 不新增后端接口，不修改权限模型。

## 方案
- 邮箱管理列表操作列改为“单入口操作菜单”：
  - 统一收纳 `拉取`、`处理`、`编辑邮箱`、`查看原始邮件`
  - 表格内只保留一个稳定宽度的 `操作` 按钮
- 操作列设置稳定宽度，避免单元格内容无限扩张。
- 保留启用状态控制：
  - 邮箱非 `enabled` 时，`拉取` 和 `处理` 仍保持禁用。

## 前后端任务拆分
- 前端
  - 调整邮箱管理列表操作列渲染方式。
  - 将所有操作统一收纳到下拉菜单。
  - 保持原有跳转、编辑和任务触发行为不变。
- 后端
  - 无代码变更，继续复用现有邮箱接口。

## 验收标准
- 邮箱管理页列表在常规桌面宽度下不再因操作列按钮过多而撑破布局。
- 用户仍能访问原有 4 个操作。
- `enabled` / `disabled` 状态下按钮可用性与原逻辑一致。
- 前端构建通过，目标文件 lint 无错误。

## 测试计划
- 打开邮箱管理页，确认操作列不再超出表格。
- 验证下拉菜单中的 `拉取`、`处理`、`编辑邮箱`、`查看原始邮件` 可正常触发。
- 验证 `拉取`、`处理` 的禁用逻辑正确。
- 执行前端 `build` 和目标文件 `eslint`。

## 风险点
- 用户所有操作都需要多一次点击，但换来稳定布局和更强扩展性。
- 如果后续邮箱管理继续增加操作项，仍可继续沿用单入口菜单，无需再次调整表格宽度。

## 待确认事项
- 当前优先保证列表稳定，不再保留任何直出操作按钮；如果你希望保留 1 个最高频动作为直出，可再细调。
