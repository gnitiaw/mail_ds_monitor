# 功能名称：前端蓝白主题优化规范

## 一、背景
当前前端页面已经具备基础功能，但视觉层仍接近默认脚手架状态，缺少统一的主题、层次和品牌识别。为了支持后续联调与演示，需要在不改变现有业务结构的前提下，将界面统一为“蓝白配色、简洁明快、大方专业”的风格。

## 二、目标
- 建立统一的蓝白主题变量和页面视觉规范
- 提升导航、卡片、表格、表单、详情页的信息层次
- 保持简洁、专业、易读，不做花哨装饰
- 不修改业务信息结构，不引入接口契约变化

## 三、设计原则
- 主色明确：以清爽蓝为主，白色和浅灰蓝为背景基底
- 层次克制：通过留白、阴影、边框和字号形成层次，不依赖复杂装饰
- 信息优先：页面先强调数据和操作，再强调装饰
- 一致性：所有页面使用同一套色板、圆角、间距、按钮和状态规范
- 易维护：尽量通过 CSS 变量和 antd theme token 管理主题，不在页面内散落硬编码颜色

## 四、主题配色规范
### 主色
- Primary：`#1677FF`
- Primary Hover：`#4096FF`
- Primary Active：`#0958D9`

### 辅助色
- Secondary Blue：`#69B1FF`
- Light Blue Surface：`#F0F7FF`
- Accent Blue Border：`#B7D7FF`

### 中性色
- Page Background：`#F5F9FF`
- Container Background：`#FFFFFF`
- Header Background：`#FFFFFF`
- Sidebar Background：`#F7FBFF`
- Border Color：`#DCE9F7`
- Divider Color：`#EAF2FB`
- Text Primary：`#16324F`
- Text Secondary：`#5B7594`
- Text Tertiary：`#8AA0BA`

### 状态色
- Success：`#31A46C`
- Warning：`#F5A623`
- Error：`#E14D4D`
- Info：`#1677FF`

## 五、字体与排版
- 中文优先字体：`"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif`
- 正文字号：`14px`
- 页面主标题：`24px / 600`
- 模块标题：`18px / 600`
- 卡片小标题：`16px / 600`
- 辅助文字：`13px`
- 行高：
  - 正文：`1.6`
  - 表格与表单：`1.5`

## 六、圆角、阴影与间距
- 基础圆角：`12px`
- 大卡片圆角：`16px`
- 输入框/按钮圆角：`10px`
- 主内容卡片阴影：`0 10px 30px rgba(22, 50, 79, 0.06)`
- 悬浮阴影：`0 14px 36px rgba(22, 50, 79, 0.10)`
- 页面外边距：`24px`
- 模块间距：`16px`
- 表单项纵向间距：`16px`

## 七、布局规范
### 整体布局
- 页面背景使用浅灰蓝，不使用纯白整页铺满
- 左侧导航保持固定宽度，建议 `220px`
- 顶部 Header 保持简洁，用于页面标题、用户入口、环境标识等
- 内容区采用“页面标题区 + 工具栏 + 主卡片”三段结构

### 左侧导航
- 背景使用浅蓝白色，不使用纯黑透明块
- Logo 区建议使用渐变蓝底 + 白字
- 当前选中菜单使用浅蓝背景、蓝色文字、轻边框
- 未选中菜单保持低干扰中性色

### 内容区
- 页面标题区包含：
  - 页面标题
  - 简短说明文案
  - 右侧主操作按钮
- 列表、详情、表单等核心内容应放入白底卡片中

## 八、组件规范
### 按钮
- 主按钮：蓝底白字，hover 更亮
- 次按钮：白底蓝边蓝字
- 危险按钮：仅用于明确危险操作，首期尽量少用
- 列表中的文字按钮颜色保持统一，不混用多个强调色

### 表单
- Label 使用中深蓝灰色
- 输入框聚焦边框统一主蓝色
- 表单错误提示使用标准 error 色
- 弹窗表单宽度适中，避免过窄

### 表格
- 表头背景使用极浅蓝灰
- 行 hover 使用轻微蓝色高亮
- 状态 Tag 颜色统一：
  - `pending`：蓝灰
  - `parsed`：信息蓝
  - `archived`：成功绿
  - `failed`：错误红
- 表格外层必须放在卡片中，不裸露在页面背景上

### 卡片
- 页面核心内容均使用白底卡片
- 卡片标题与正文之间留足间距
- 详情页卡片建议支持分区展示，而不是大段堆叠

### 空态与加载态
- 空态保持简洁，不要使用默认空白占位就结束
- 加载态优先使用 skeleton 或卡片级 spin，不要整页闪烁

## 九、页面级设计要求
### 邮箱管理页
- 顶部使用“标题 + 描述 + 新增按钮”
- 表格卡片显示邮箱信息和操作
- 新增/编辑弹窗统一样式，不要使用默认密集布局

### 归档列表页
- 筛选区单独放在顶部卡片
- 结果列表放在主卡片
- 主题列可点击，并有清晰 hover 提示
- 提取状态和置信度在视觉上形成组合，不要散乱

### 归档详情页
- 建议拆为三张卡片：
  - 基础邮件信息
  - AI 提取结果
  - 邮件正文
- AI 失败时的错误提示使用红色 Alert，但整体布局仍保持克制

### 汇总配置页
- 列表页保持简洁，突出配置名称、发送时间、收件人、状态
- 新建配置弹窗要有更清晰的分组，例如“发送规则”“汇总范围”“状态策略”

### 汇总发送记录页
- 筛选区与结果区分离
- `pending`、`success`、`failed` 状态颜色必须统一
- 失败原因查看入口要明显但不过度抢占视觉

## 十、实现建议
- 优先使用 `ConfigProvider` 的 antd theme token 做全局主题
- 在 `src/index.css` 中定义 CSS 变量，例如：
  - `--color-primary`
  - `--color-page-bg`
  - `--color-card-bg`
  - `--color-text-primary`
  - `--shadow-card`
- 页面尽量少写内联样式，将布局与主题抽到公共 class
- 如需新增公共样式文件，建议放在 `src/styles/`

## 十一、验收标准
- [ ] 所有页面统一使用蓝白主题，不再出现默认脚手架风格
- [ ] 左侧导航、Header、内容卡片、表格、弹窗形成统一视觉语言
- [ ] 页面层级清晰，主操作按钮突出但不过度抢眼
- [ ] 表格、筛选区、详情区有明确分区和留白
- [ ] 颜色、圆角、阴影、边框、字号使用统一规范
- [ ] 不改变现有接口契约和业务字段含义

## 十二、风险清单
- 如果主题变量和页面样式没有统一收敛，后续页面会继续出现样式漂移
- 如果过度依赖内联样式，后期维护成本会变高
- 若直接覆盖过多 antd 默认样式，可能影响表单和表格可用性

## 十三、待确认问题
- 顶部 Header 是否需要预留用户信息、环境标识或刷新入口
- 是否需要在首屏加入系统欢迎区或概览卡片
- 是否需要在移动端做同等优先级适配，还是首期以桌面端为主
